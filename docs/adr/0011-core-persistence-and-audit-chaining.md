# ADR 0011: Core persistence model and audit chaining

- Status: Accepted
- Date: 2026-06-18
- Deciders: Metis maintainers

## Context

`metis-core` is the first real implementation of the protocol storage interfaces.
It must enforce the truth hierarchy and the cross-stage invariants (raw immutability,
derived versioning, append-only memory, audit-on-write) at the storage boundary, on
PostgreSQL + S3/MinIO. Stage 2 left three things to settle here: the row<->model
storage shape, how the per-workspace audit hash chain stays linear under concurrency,
and the initial-migration mechanics.

## Decision

**Storage shape — body JSONB + denormalized columns.** Each row stores the full
protocol model dump in a `body` JSONB column and reconstructs via
`model_validate(body)` (translate-on-read, ADR 0010), plus a small set of extracted,
indexed columns for tenancy, FKs, and filtering (`id`, `workspace_id`, `sensitivity`,
`created_at`, `tombstoned_at`, content hash, type-specific keys). FKs encode the
evidence chain where callers control write order (doc←artifact, parsed←doc,
segment←parsed, span→artifact); cross-references that must tolerate forward/superseded
targets or synthetic ids from the contract suites (extraction-batch `parsed_doc_id`,
memory `scene`/`supersedes`/`target`, wiki `page_id`, claim source-span refs) are plain
columns. This keeps mappers trivial and additive schema changes cheap.

**Audit chaining — advisory lock per workspace.** The audit log is append-only and
hash-chained per workspace: `audit_hash = sha256(prev_hash || seq || canonical_event)`,
with a monotonic per-workspace `seq`. Appends serialize on a transaction-scoped
`pg_advisory_xact_lock(hashtext(workspace_id))`, so the chain is never racy, and the
audit row is written **inside the store's transaction** — an audit event is atomic with
the write it records. `verify_chain` replays a workspace's rows and reports the first
break (bad seq, prev-hash, or content hash).

**Immutability / append-only at the store layer.** Raw artifacts dedup on
`(workspace_id, content_hash)` and there is no update path: a re-put returns the existing
ref. Memory `supersede`/`retract` flip flags on the target row (it stays for audit) and
are hidden from default reads. Tombstoning a raw artifact cascades to derived docs,
segments, claims (via the artifact ids in span refs), and mem cells; rows are marked,
not deleted, and hidden from reads.

**Infra interfaces** (`ObjectStore`, `JobQueue`) remain protocol contracts (ADR 0010)
and are implemented here. The job queue uses `FOR UPDATE SKIP LOCKED`.

**Initial migration via `metadata.create_all`.** `0001_initial` creates the pgvector
extension, then builds all tables from the ORM metadata (models are the Phase-0 source
of truth); later migrations are hand-written/autogenerated. The reserved `embedding`
column is dimensionless and unindexed until Stage 8 fixes an embedding model.

**Test infra — testcontainers.** Contract and store tests run against ephemeral
Postgres (pgvector image) + MinIO; the suite skips cleanly when no Docker daemon is
present, so the pure policy tests still run.

## Consequences

- Stores pass the Stage 1 contract suites unchanged against real engines.
- The audit chain is tamper-evident and ready for Stage 14 integrity checks.
- A single per-workspace serialization point (the advisory lock) bounds audit-append
  throughput per workspace; acceptable for Phase 0, revisit if it becomes hot.
- `body` duplicates indexed columns; writes derive both from one model, so they stay
  consistent. Queryability beyond the extracted columns goes through JSONB operators.
- CI must expose a Docker daemon for the testcontainers suite (flagged in Stage 0).

## Alternatives considered

- **Fully normalized columns per field** instead of `body` JSONB: richer SQL queries,
  but heavy schema churn as protocol schemas grow, and a fatter mapper. Rejected for
  Phase 0; selective promotion of hot fields to columns stays open.
- **Monotonic sequence with deferred chaining** instead of an advisory lock: avoids a
  lock but needs a separate serialization/repair step. The advisory lock is simpler and
  keeps appends synchronous and atomic.
- **Per-job Redis/Celery queue**: more capable, but new infra; the `JobQueue` interface
  keeps that swap cheap later.
